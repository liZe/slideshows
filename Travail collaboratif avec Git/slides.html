Présentation du problème

Les contraintes du développement

On écrit majoritairement des fichiers avec du texte brut.

On ajoute des images et quelques autres fichiers.

On travaille en équipe.

On veut garder trace de toutes les versions.

Ce qu'on ne veut pas

Faire des changements incompatibles avec ceux du voisin.

Avoir des versions 1, 2, 2.1, 2.1-bis, jeudi-dernier-dans-la-matinée…

Perdre notre précieux travail.

Bonne nouvelle

Des gens sympas ont déjà fait ce qu'il faut pour nous.

Un peu d'histoire

Ça commence il y a très longtemps

Dès les années 70, on crée des outils pour garder trace de l'évolution des projets (SCCS en 1972, RCS en 1982).

On cherche ensuite à partager en ligne les projets (CVS en 1990, Subversion en 2000).

La génération suivante cherche à décentraliser le développement (BitKeeper en 2000, Git et Mercurial en 2005).

Aujourd'hui

Git est de très loin l'outil de gestion de versions le plus utilisé aujourd'hui.

D'autres outils sont également utilisés, souvent pour des usages particuliers : Mercurial, Subversion, Perforce…

GitHub

GitHub est le service d'hébergement de dépôts Git le plus utilisé dans le monde. D'autres outils et services existent : GitLab, BitBucket…

GitHub Desktop

GitHub Desktop est un logiciel permettant de gérer les projets sur GitHub. C'est un outil simple pour créer de nouvelles versions.
