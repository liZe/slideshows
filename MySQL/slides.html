Présentation

SGBDR

MySQL est un système de gestion de bases de données relationnelles.

Le nom MySQL vient de la fille de Michael Widenius, My.

Historique

MySQL a été créé en 1995 par David Axmark, Allan Larsson et Michael Widenius, au sein de la société suédoise MySQL AB.

L’entreprise a été rachetée pour 1 milliard de dollars par Sun Microsystems en 2008, elle-même rachetée par Oracle en 2010.

MariaDB

Suite au rachat de Sun Microsystems par Oracle, Michael Widenius a forké le projet et créé MariaDB.

Licence

MySQL est un logiciel libre distribué sous licence GPLv2.

Il est également vendu sous licence propriétaire par Oracle.

Code

MySQL est écrit majoritairement en C et C++.

Le serveur fonctionne sur une grande liste de plateformes (Linux, macOS, Windows, BSD, Solaris, AIX, HP-UX…).

Utilisations

Popularité

MySQL est l’un des gestionnaires de bases de données les plus utilisés dans le monde.

MySQL est la base de données de référence pour beaucoup de frameworks web (WordPress, Drupal, Joomla…). Elle est largement utilisée par les plus grands acteurs du web (Facebook, Wikipedia, Airbnb, Booking, YouTube, Spotify, Netflix…).

Simplicité

La grande popularité de MySQL réside avant tout dans sa simplicité d’utilisation, dans l’abondance de sa documentation et dans ses performances à grande échelle.

MySQL est l’une des composantes de l’architecture historique LAMP (Linux, Apache, MySQL, PHP), démocratisée dans les années 2000 pour l’installation de serveurs web.

Interfaces

Un grand nombre d’interfaces pour administrer et utiliser les bases de données ont été développées pour MySQL :

  • une interface en ligne de commande (mysql),
  • des clients lourds (MySQL Workbench, DBeaver, LibreOffice Base…),
  • des clients web (phpMyAdmin…),
  • des connecteurs pour un grand nombre de langages de programmation (Java, Python, JavaScript, PHP…).

Fonctionnalités

SQL

MySQL offre un large support du langage SQL, bien qu’elle n’en suive pas strictement la syntaxe.

De nombreuses extensions au langage ont également été apportées.

Moteurs

MySQL supporte différents moteurs de bases de données. Les deux principaux sont MyISAM et InnoDB.

MyISAM

MyISAM est le moteur historique de MySQL, et en était le moteur par défaut jusqu’en 2010.

Il est particulièrement performant pour l’accès aux données en lecture seule.

Il ne supporte pas les transactions, ni la vérification des contraintes de clés étrangères.

InnoDB

Le moteur InnoDB a été créé pour pallier les limitations de MyISAM. Il est aujourd’hui installé par défaut dans la plupart des versions de MySQL proposées par Oracle.

Il supporte les transactions (au sens ACID du terme) et les contraintes de clés étrangères.

Performances

MySQL est reconnu pour offrir un très haut niveau de performances. En faisant le choix de prendre certaines libertés avec la norme SQL, elle a pu s’affranchir de certaines contraintes et privilégier la rapidité d’exécution.

Scalabilité

MySQL est très largement utilisée par des acteurs gérant d’immenses quantités de données.

La base de données propose des outils permettant d’assurer le stockage de volumes importants sur des architectures distribuées.

Pour aller plus loin

Complémentarité

MySQL apporte des avantages différents de ceux des autres SGBDR libres, en particulier PostgreSQL et SQLite. Cette complémentarité donne à l’ensemble de ces solutions une immense adoption par les utilisateurs, et une satisfaction globale relativement élevée.

Oracle

Le rachat de Sun par Oracle a créé un grand émoi dans la communauté qui développe MySQL. La création de MariaDB, qui a emporté avec elle une partie importante des développeurs, montre la méfiance d’une partie des utilisateurs vis-à-vis de l’entreprise américaine reconnue sur le marché des bases de données.

NoSQL

Les bases de données non relationnelles existent depuis longtemps, mais une vague de nouveaux outils a émergé au début de ce siècle (REDIS, CouchDB, MongoDB, ElasticSearch…)

Ces bases de données NoSQL proposent de nombreux avantages : stockage sans schéma, stockage clé-valeur, vitesse, distribution… Il faut cependant noter qu’en perdant la rigidité du côté relationnel, elles en perdent souvent les avantages, et en particulier la cohérence des données.
